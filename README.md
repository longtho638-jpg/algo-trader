

## Usage Guide

### Running Backtests

To test a strategy against historical data without risking real capital:

```bash
# Ensure ENABLE_BACKTESTING is set to true in your .env file.
# Configure the strategy parameters (e.g., TRADING_PAIR, TIMEFRAME) in .env or config files.

# Build the project
npm run build

# Execute a backtest (example for RsiSmaStrategy)
npm run backtest --strategy=RsiSmaStrategy --pair=BTC/USDT --timeframe=1h
# Output will be generated by the configured reporters (e.g., console, HTML report)
```

### Running Live Trading

To deploy the bot for live trading on an exchange:

```bash
# Ensure ENABLE_LIVE_TRADING is set to true in your .env file.
# Verify your EXCHANGE_API_KEY and EXCHANGE_SECRET are correctly set in .env.
# Double-check all trading and risk management parameters.

# Build the project
npm run build

# Start live trading (example for RsiSmaStrategy)
npm start --strategy=RsiSmaStrategy --pair=BTC/USDT --timeframe=1h
# The bot will connect to the exchange and begin trading based on the strategy.
# Monitor its activity via the CLI dashboard or configured logging.
```

**IMPORTANT:** Live trading involves real financial risk. Thoroughly test your strategies with backtests and paper trading before deploying to a live account.

### arb:spread (Cross-Exchange Spread Detector)

```bash
# Default: BTC/USDT + ETH/USDT on Binance/OKX/Bybit, auto-execute
npx ts-node src/index.ts arb:spread

# Dry-run (scan only, no trades)
npx ts-node src/index.ts arb:spread --dry-run

# Custom config
npx ts-node src/index.ts arb:spread \
  -p BTC/USDT,ETH/USDT \
  -e binance,okx,bybit \
  -s 500 \
  -t 0.08 \
  --score-threshold 60 \
  --max-loss 50

# Limited polls (stop after 100 cycles)
npx ts-node src/index.ts arb:spread --max-polls 100
```

**Features:**
- Real-time spread dashboard every 5 cycles
- Signal scoring + orderbook validation (auto-exec mode)
- Circuit breaker with max daily loss protection
- `--dry-run` for safe monitoring without execution

### arb:agi (Recommended for Production)

```bash
# Default: full AGI pipeline with regime detection + Kelly sizing
npx ts-node src/index.ts arb:agi

# Custom config
npx ts-node src/index.ts arb:agi \
  -p BTC/USDT,ETH/USDT \
  -e binance,okx,bybit \
  -s 1000 \
  --score-threshold 65 \
  --max-loss 100

# Disable optional AGI features
npx ts-node src/index.ts arb:agi --no-regime --no-kelly --no-self-tune

# Custom regime detection interval (ms)
npx ts-node src/index.ts arb:agi --regime-interval 15000
```

**AGI Features:**
- **Regime Detection** вЂ” Hurst exponent + volatility ratio classifies market (trending/mean-reverting/volatile/quiet)
- **Kelly Position Sizing** вЂ” Optimal position sizing based on historical win rate and payoff ratio
- **Self-Tuning** вЂ” Score/spread thresholds auto-adjust via EMA of profitability
- **Strategy Routing** вЂ” Execution parameters adapt per detected regime
